# Plan 04-03: Notifications & Navigation Integration

## Objective

Complete the lost pets module by adding the match notifications page where users see alerts when their reports match newly submitted reports, and integrate Lost Pets navigation into the shared layout. This covers LOST-03 (user receives basic alerts for matching lost pets). This plan adds the Notifications action to the existing Customer area LostPetController (created in Plan 04-02), creates the Notifications view, and adds Lost Pets links to the site navigation.

**Purpose:** Delivers the final piece of the lost pets module — the match discovery experience that creates community-driven pet recovery. Users who reported a lost or found pet can see when a matching report is submitted.

**Output:** Notifications page showing match alerts with read/unread state, Lost Pets accessible from main navigation.

## Tasks

### Wave 3A — Notifications Controller Actions

- **type**: edit
  **files**: [src/PetPlatform.Host.MVC/Areas/Customer/Controllers/LostPetController.cs]
  **action**: Add two new actions to the existing Customer area `LostPetController` (created in Plan 04-02):

    - `Notifications()` [GET]:
      - Get userId. If null, return Challenge()
      - Call `_lostPetService.GetMyNotificationsAsync(userId)`
      - Return View with `IEnumerable<MatchNotificationDto>`

    - `MarkNotificationRead(int notificationId)` [POST]:
      - `[ValidateAntiForgeryToken]`
      - Get userId. If null, return Challenge()
      - Call `_lostPetService.MarkNotificationReadAsync(notificationId, userId)`
      - If success: `TempData["Success"] = "Notification marked as read."`
      - If failure: `TempData["Error"] = result.Error`
      - RedirectToAction("Notifications")
  **verify**: Controller compiles with 2 new actions (total 9 actions), follows existing pattern
  **acceptance_criteria**: Notifications action returns all user's notifications. MarkNotificationRead is POST-only with antiforgery. Ownership checked by service layer (ReporterUserId). TempData feedback on mutations.
  **read_first**: [src/PetPlatform.Host.MVC/Areas/Customer/Controllers/LostPetController.cs]

### Wave 3B — Notifications View

- **type**: create
  **files**: [src/PetPlatform.Host.MVC/Areas/Customer/Views/LostPet/Notifications.cshtml]
  **action**: Create Razor view for match notifications page:
    - Model: `IEnumerable<MatchNotificationDto>`
    - **Header**: "Match Notifications" with unread count badge (if any unread)
    - **Notifications list** (card-based layout):
      - Each notification card shows:
        - **Match indicator icon**: 🔔 for unread, ✔️ for read (or use Tailwind color: unread=blue border, read=gray border)
        - **Message**: The notification text describing the match
        - **Matched report summary**: MatchedReportSpecies, MatchedReportLocation, MatchedReportDescription (truncated to 200 chars)
        - **Timestamp**: CreatedAt (relative time format like "2 hours ago" or formatted date)
        - **Action**: "View Matched Report" link → navigates to public `/LostPet/Details/{MatchedReportId}` (so user can see the report that matched theirs)
        - **Mark as Read button** (POST form with antiforgery): Only shown if IsRead == false. Button text: "Dismiss"
      - Cards are sorted by CreatedAt descending (newest first, from service layer)
    - **Empty state**: "No match notifications yet. When a pet report matches yours, you'll see it here." with link back to "My Reports" and "Report a Pet"
    - **Success/Error messages**: Display TempData messages
    - **Visual distinction**: Unread notifications have a subtle blue left border or highlight. Read notifications have a gray/neutral appearance. Per D-07, notifications are visible only to the reporter — this view requires auth.
  **verify**: View renders notification cards with correct fields, conditional Mark as Read button, empty state
  **acceptance_criteria**: All notifications displayed with message, matched report summary, timestamp, and link. Unread/Read visual distinction. Mark as Read POST button only for unread. Empty state guides user. Auth required (inherits from controller [Authorize]).
  **read_first**: [src/PetPlatform.Host.MVC/Areas/Customer/Controllers/LostPetController.cs]
