# Plan 04-02: Customer-Facing Reporting & Search UI

## Objective

Build the public-facing lost pet search experience and the authenticated customer's report creation workflow. Users can browse/search all open lost pet reports with filters (report type, species, breed, color, location, date range), view report details, create new lost/found pet reports with photo uploads (1-5 photos), edit their own open reports, and manage their reports. This covers LOST-01 (report lost/found pets with details) and LOST-02 (search by species, breed, color, location, date).

**Purpose:** Delivers the core user-facing experience for the lost pets module — the ability to find and report pets.

**Output:** Fully functional public search page, report detail page, report creation form with multi-photo upload, report edit form, and my-reports dashboard.

### Wave 2C — Customer Area LostPetController (Create/Edit/Manage)

- **type**: create
  **files**: [src/PetPlatform.Host.MVC/Areas/Customer/Controllers/LostPetController.cs]
  **action**: Create Customer area `LostPetController` following `AdoptionController.cs` and `MyAccountController.cs` patterns:
    - `[Area("Customer")]`, `[Authorize]`
    - Constructor: inject `ILostPetService`
    - Helper: `GetUserId()` private method extracting `User.FindFirst(ClaimTypes.NameIdentifier)?.Value`, returning null if empty (follow MyAccountController pattern)

    **Actions:**

    - `MyReports()` [GET]:
      - Get userId. If null, return Challenge()
      - Call `_lostPetService.GetMyReportsAsync(userId)`
      - Return View with `IEnumerable<LostPetReportDto>`

    - `Create()` [GET]:
      - Get userId. If null, return Challenge()
      - Create empty `CreateLostPetReportDto` with `DateReported = DateTime.Today`
      - Return View with dto

    - `Create(CreateLostPetReportDto dto, List<IFormFile> photos)` [POST]:
      - `[ValidateAntiForgeryToken]`
      - Get userId. If null, return Challenge()
      - Call `_lostPetService.CreateReportAsync(dto, userId, photos)`
      - If success: `TempData["Success"] = "Your lost pet report has been submitted! We're checking for matches..."` → RedirectToAction("ReportDetails", new { id = result.Value.Id })
      - If failure: `ModelState.AddModelError(string.Empty, result.Error)` → return View(dto)
      - Per D-08, the matching check message hints that matches were processed synchronously

    - `ReportDetails(int id)` [GET]:
      - Get userId. If null, return Challenge()
      - Call `_lostPetService.GetReportByIdAsync(id)` (public, but we also need to verify ownership for edit/resolve buttons)
      - Call `_lostPetService.GetMyReportsAsync(userId)` to check if this report belongs to current user
      - If report not found: return NotFound()
      - Set `ViewBag.IsOwner = true/false` based on ReporterUserId match
      - Set `ViewBag.ReportStatus` for conditional UI
      - Return View with `LostPetReportDto`

    - `Edit(int id)` [GET]:
      - Get userId. If null, return Challenge()
      - Call `_lostPetService.GetReportByIdAsync(id)`
      - If not found: return NotFound()
      - Verify report belongs to current user (ReporterUserId == userId), else return Forbid()
      - Verify report.Status == Open (D-15), else return Forbid() with TempData error
      - Map to `UpdateLostPetReportDto` (Color, Breed, Location, DateReported, Description)
      - Pass existing photos to ViewBag for display
      - Return View with dto

    - `Edit(int id, UpdateLostPetReportDto dto, List<IFormFile> photos)` [POST]:
      - `[ValidateAntiForgeryToken]`
      - Get userId. If null, return Challenge()
      - Call `_lostPetService.UpdateReportAsync(id, dto, userId, photos)`
      - If success: `TempData["Success"] = "Report updated successfully."` → RedirectToAction("ReportDetails", new { id })
      - If failure: `ModelState.AddModelError(string.Empty, result.Error)` → return View(dto)

    - `Resolve(int id)` [POST]:
      - `[ValidateAntiForgeryToken]`
      - Get userId. If null, return Challenge()
      - Call `_lostPetService.ResolveReportAsync(id, userId)`
      - If success: `TempData["Success"] = "Report marked as resolved."`
      - If failure: `TempData["Error"] = result.Error`
      - RedirectToAction("ReportDetails", new { id })
  **verify**: Controller compiles, follows AdoptionController and MyAccountController patterns (userId extraction, Challenge(), Result handling)
  **acceptance_criteria**: All 7 actions present. GET Create pre-fills DateReported with today. POST Create has ValidateAntiForgeryToken. POST Edit has ValidateAntiForgeryToken. Resolve is POST-only. Edit checks ownership AND Open status before showing form. ReportDetails passes ownership info to ViewBag for conditional UI.
  **read_first**: [src/PetPlatform.Host.MVC/Areas/Customer/Controllers/AdoptionController.cs, src/PetPlatform.Host.MVC/Areas/Customer/Controllers/MyAccountController.cs]
