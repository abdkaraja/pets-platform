Build success URL: `Url.Action("Confirmation", "Checkout", new { session_id = "{CHECKOUT_SESSION_ID}" }, Request.Scheme)` (per RESEARCH Pattern 5). **checkout.js (src/PetPlatform.Host.MVC/wwwroot/js/checkout.js):**
Per RESEARCH Pattern 5 (client-side Stripe.js):
- On DOMContentLoaded: read publishable key and client secret from data attributes or hidden inputs (set by Razor from ViewData)
- Initialize Stripe: `const stripe = Stripe(publishableKey)`
- Create elements: `const elements = stripe.elements({ clientSecret: clientSecret, appearance: { theme: 'stripe' } })`
- Create and mount payment element: `const paymentElement = elements.create('payment'); paymentElement.mount('#payment-element')`
- Form submit handler: prevent default, disable button, show "Processing...", call `stripe.confirmPayment({ elements, confirmParams: { return_url: returnUrl } })`. On error, show error message in error display area, re-enable button.
- The form POSTs to CreateSession via AJAX first to get the session, then Stripe.js confirms payment client-side.

Flow: 
1. Page loads → checkout.js calls POST /Checkout/CreateSession via fetch
2. Server returns { sessionId, clientSecret, publishableKey }
3. checkout.js initializes Stripe and Payment Element
4. User enters card details and clicks Pay
5. stripe.confirmPayment sends card data to Stripe (never to our server)
6. On success: redirect to /Checkout/Confirmation?session_id={id}
7. Stripe webhook confirms payment server-side

**Program.cs webhook endpoint:**
Add the webhook endpoint registration. Since the webhook must NOT require authentication (Stripe calls it externally), add it as a minimal endpoint BEFORE the auth middleware or as a separate route:

In the endpoint configuration section, add:
```csharp
app.MapPost("/webhook", async (HttpContext context, IPaymentService paymentService, IConfiguration config) =>
{
    var json = await new StreamReader(context.Request.Body).ReadToEndAsync();
    try
    {
        var stripeEvent = EventUtility.ConstructEvent(
            json,
            context.Request.Headers["Stripe-Signature"].FirstOrDefault(),
            config["Stripe:WebhookSecret"]);
        if (stripeEvent.Type == Events.CheckoutSessionCompleted)
        {
            var session = stripeEvent.Data.Object as Stripe.Checkout.Session;
            if (session != null)
                await paymentService.HandleCheckoutSessionCompletedAsync(session.Id);
        }
        return Results.Ok();
    }
    catch (StripeException)
    {
        return Results.BadRequest();
    }
}).AllowAnonymous();
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → CheckoutController | Shipping info and payment initiation — validated server-side |
| Browser → Stripe (Payment Element) | Card data handled by Stripe.js — never touches our server |
| Stripe → /webhook | Webhook events must be signature-validated — critical security boundary |
| User → OrderController | User-scoped order queries — must verify ownership |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-2-12 | Tampering | Stripe webhook signature spoofing | critical | mitigate | Validate Stripe-Signature header with EventUtility.ConstructEvent() using webhook secret from config |
| T-2-13 | Tampering | Client-side price manipulation on checkout | high | mitigate | Server creates Stripe session with prices from DB (LockedPrice); client never submits prices |
| T-2-14 | Information Disclosure | Stripe secret key exposure | critical | mitigate | Only PublishableKey sent to client; SecretKey stays in server config; checkout.js uses Stripe(publishableKey) only |
| T-2-15 | Tampering | CSRF on checkout form submission | medium | mitigate | ASP.NET Core anti-forgery tokens; AJAX calls include request verification token |
| T-2-16 | Information Disclosure | Order data leakage across users | medium | mitigate | OrderController passes UserId to OrderService; service verifies ownership before returning data |
| T-2-17 | Denial of Service | Webhook endpoint abuse | low | mitigate | Webhook validates Stripe signature; rejects all requests without valid signature |
</threat_model>
