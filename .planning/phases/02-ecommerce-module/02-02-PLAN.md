**Step 3: Create Admin ProductController**
Create `src/PetPlatform.Host.MVC/Areas/Admin/Controllers/ProductController.cs`:
- Namespace: `PetPlatform.Host.MVC.Areas.Admin.Controllers`
- Attributes: `[Area("Admin")]`, `[Authorize(Policy = "Permission:Products.Manage")]`
- Inject: `IProductService`, `ICategoryService`
- Actions:
  - `Index(string? searchTerm, int? categoryId, int page = 1)` → GET `/Admin/Products` — calls `_productService.GetAllAdminAsync(searchTerm, categoryId)` with paging. Returns View with products.
  - `Create()` → GET `/Admin/Products/Create` — loads categories and brands for dropdowns. Returns View.
  - `Create(AdminProductDto dto, IFormFile? imageFile)` → POST — validates, saves image to wwwroot/uploads/products/ if provided, calls `_productService.CreateAsync(dto)`. Redirect to Index on success.
  - `Edit(int id)` → GET `/Admin/Products/Edit/{id}` — loads product with variants for editing. Returns View.
  - `Edit(int id, AdminProductDto dto, IFormFile? imageFile)` → POST — validates, handles image upload/replacement, calls `_productService.UpdateAsync(id, dto)`. Redirect to Index.
  - `Delete(int id)` → POST — calls `_productService.DeleteAsync(id)`. Redirect to Index.
- Use `[ValidateAntiForgeryToken]` on all POST actions.

**Step 3: Create CategoryController**
File: `src/PetPlatform.Host.MVC/Areas/Admin/Controllers/CategoryController.cs`
- `[Area("Admin")]`, `[Authorize(Policy = "Permission:Categories.Manage")]`
- Inject: `ICategoryService`
- Actions:
  - `Index()` → GET `/Admin/Categories` — calls `_categoryService.GetTreeAsync()`. Returns View with hierarchical categories.
  - `Edit(int? id)` → GET — if id provided, loads existing category; otherwise new. Returns View.
  - `Edit(int id, CreateCategoryDto dto)` → POST with `[ValidateAntiForgeryToken]` — if id == 0, create; else update. Redirect to Index.
  - `Delete(int id)` → POST with `[ValidateAntiForgeryToken]` — calls DeleteAsync. Redirect to Index.

**Step 4: Create InventoryController**
File: `src/PetPlatform.Host.MVC/Areas/Admin/Controllers/InventoryController.cs`
- `[Area("Admin")]`, `[Authorize(Policy = "Permission:Inventory.Manage")]`
- Inject: `IInventoryService`
- Actions:
  - `Index(string? searchTerm, string? stockFilter)` → GET `/Admin/Inventory` — calls `_inventoryService.GetAllVariantsAsync(searchTerm, stockFilter)`. Returns View.
  - `UpdateStock(int variantId, int newQuantity)` → POST (AJAX JSON endpoint) — calls `_inventoryService.UpdateStockAsync(variantId, newQuantity)`. Returns JSON result `{ success, message, newStock, status }`.
- The UpdateStock action should accept JSON body `[FromBody] UpdateStockRequest request` with variantId and newQuantity.

**Step 5: Create OrderManagementController**
File: `src/PetPlatform.Host.MVC/Areas/Admin/Controllers/OrderManagementController.cs`

- Inject: `IOrderService`

- Add a private helper method `GetNextAllowedStatus(OrderStatus current)` that returns the next valid status in the workflow (Pending→Processing, Processing→Shipped, Shipped→Delivered, Delivered→null).

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Admin form → Controller | Product/category form submissions must be validated server-side |
| Admin status change → Order | Order status changes must go through policy authorization |
| Inline stock edit → Inventory | AJAX stock updates must be admin-authorized and validated |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-2-08 | Elevation of Privilege | Admin product/category/order endpoints | high | mitigate | All controllers require Admin role with specific permission claims via policy-based authorization |
| T-2-09 | Tampering | Product price manipulation via form | medium | mitigate | Price validated server-side by FluentValidation; BasePrice > 0 enforced |
| T-2-10 | Tampering | Inline stock update to negative values | medium | mitigate | InventoryService.UpdateStockAsync validates quantity >= 0; returns error for negative |
| T-2-11 | Tampering | Order status skip (e.g., Pending→Delivered) | high | mitigate | OrderManagementController uses GetNextAllowedStatus helper; dropdown only shows next valid status; Order.UpdateStatus validates transition |
</threat_model>
