# Plan 03-03: Shelter Management (Listings CRUD + Application Review)

## Objective

Build the ServiceProvider area's shelter management interface. Shelter operators can create and manage adoption listings (create, edit, close), view applications per listing, and review (approve/reject) applications with notes. This covers ADPT-04 (shelter review/approve/reject) and ADPT-05 (status updates for shelter-side). This plan depends on both Plan 01 (service layer) and Plan 02 (customer-facing flows exist to test against).

### Wave 2B — Shelter Controller

- **type**: create
  **files**: [src/PetPlatform.Host.MVC/Areas/ServiceProvider/Controllers/AdoptionController.cs]
  **action**: Create ServiceProvider area `AdoptionController` following Admin area `OrderManagementController.cs` pattern:
    - `[Area("ServiceProvider")]`, `[Authorize(Policy = "Permission:Adoptions.Manage")]`
    - Constructor: inject `IAdoptionService`
    - Helper: `GetUserId()` private method extracting `User.FindFirst(ClaimTypes.NameIdentifier)?.Value`

    **Actions:**

    - `Listings()` [GET]:
      - Get userId. If null, return Challenge()
      - Call `_adoptionService.GetShelterListingsAsync(userId)`
      - Return View with `IEnumerable<AdoptionListingDto>`

    - `CreateListing()` [GET]:
      - Return View with empty `CreateListingDto`
      - Pre-populate ViewBag with user's pets (optional: could inject IPetService to list pets owned by shelter)

    - `CreateListing(CreateListingDto dto)` [POST]:
      - `[ValidateAntiForgeryToken]`
      - Get userId. If null, return Challenge()
      - Call `_adoptionService.CreateListingAsync(dto, userId)`
      - If success: `TempData["Success"] = "Listing created."` → RedirectToAction("Listings")
      - If failure: `ModelState.AddModelError(string.Empty, result.Error)` → return View(dto)

    - `EditListing(int id)` [GET]:
      - Get userId. If null, return Challenge()
      - Fetch listing via `_adoptionService.GetListingByIdAsync(id)` — verify it exists and belongs to this shelter
      - Map to `UpdateListingDto` (Title, Description, Location)
      - Return View with dto

    - `EditListing(int id, UpdateListingDto dto)` [POST]:
      - `[ValidateAntiForgeryToken]`
      - Get userId. If null, return Challenge()
      - Call `_adoptionService.UpdateListingAsync(id, dto, userId)`
      - If success: `TempData["Success"] = "Listing updated."` → RedirectToAction("Listings")
      - If failure: `ModelState.AddModelError` → return View(dto)

    - `Applications(int listingId)` [GET]:
      - Get userId. If null, return Challenge()
      - Call `_adoptionService.GetApplicationsForListingAsync(listingId, userId)`
      - Pass listingId to ViewBag for header display
      - Return View with `IEnumerable<AdoptionApplicationDto>`

    - `ReviewApplication(int applicationId)` [GET]:
      - Get userId. If null, return Challenge()
      - Fetch application (via GetMyApplications or a dedicated lookup — for simplicity, query through GetApplicationsForListingAsync and find by Id)
      - Pass application to View
      - Return View with `AdoptionApplicationDto` and empty `ReviewApplicationDto`

    - `ReviewApplication(int applicationId, ReviewApplicationDto dto)` [POST]:
      - `[ValidateAntiForgeryToken]`
      - Get userId. If null, return Challenge()
      - Call `_adoptionService.ReviewApplicationAsync(applicationId, dto, userId)`
      - If success: `TempData["Success"] = $"Application {dto.Status}."` → RedirectToAction("Applications", new { listingId = ... })
      - If failure: `ModelState.AddModelError` → return View(dto)
  **verify**: Controller compiles, follows OrderManagementController authorization pattern, all 8 actions present
  **acceptance_criteria**: All actions require Permission:Adoptions.Manage. Ownership checked via userId. POSTs have ValidateAntiForgeryToken. TempData feedback on mutations. Status updates redirect back to Applications list
  **read_first**: [src/PetPlatform.Host.MVC/Areas/Admin/Controllers/OrderManagementController.cs, src/PetPlatform.Host.MVC/Areas/Customer/Controllers/MyAccountController.cs]
