# Plan 03-02: Customer-Facing Adoption (Browse, Search, Apply)

## Objective

Build the public-facing adoption browsing experience and the authenticated customer's adoption workflow. Users can browse/search adoptable pet listings with filters (species, breed, age, size, location), view listing details, submit adoption applications with household information and pet experience, and track their application status. This covers ADPT-01 (browse), ADPT-02 (search/filter), ADPT-03 (submit application), and ADPT-05 (status updates).

### Wave 1H — Customer Area Adoption Controller (Apply/My Applications)

- **type**: create
  **files**: [src/PetPlatform.Host.MVC/Areas/Customer/Controllers/AdoptionController.cs]
  **action**: Create Customer area `AdoptionController` following `MyAccountController.cs` pattern:
    - `[Area("Customer")]`, `[Authorize]`
    - Constructor: inject `IAdoptionService`
    - Helper: `GetUserId()` private method extracting `User.FindFirst(ClaimTypes.NameIdentifier)?.Value`, returning null if empty (follow MyAccountController pattern)

    - `MyApplications()`:
      - Get userId. If null, return Challenge()
      - Call `_adoptionService.GetMyApplicationsAsync(userId)`
      - Return View with `IEnumerable<AdoptionApplicationDto>`

    - `Apply(int listingId)` [GET]:
      - Get userId. If null, return Challenge()
      - Fetch listing via `_adoptionService.GetListingByIdAsync(listingId)` — verify it exists and is Active
      - Create `CreateApplicationDto { ListingId = listingId }`
      - Pass listing info to ViewBag for display
      - Return View with CreateApplicationDto

    - `Apply(CreateApplicationDto dto)` [POST]:
      - `[ValidateAntiForgeryToken]`
      - Get userId. If null, return Challenge()
      - Call `_adoptionService.SubmitApplicationAsync(dto, userId)`
      - If success: `TempData["Success"] = "Your application has been submitted!"` → RedirectToAction("ApplicationDetails", new { id = result.Value.Id })
      - If failure: `ModelState.AddModelError(string.Empty, result.Error)` → return View(dto)

    - `ApplicationDetails(int id)` [GET]:
      - Get userId. If null, return Challenge()
      - Call `_adoptionService.GetMyApplicationsAsync(userId)` — filter to find application with matching id
      - If not found or doesn't belong to user: return NotFound()
      - Return View with `AdoptionApplicationDto`

    - `Withdraw(int id)` [POST]:
      - `[ValidateAntiForgeryToken]`
      - Get userId. If null, return Challenge()
      - Call `_adoptionService.WithdrawApplicationAsync(id, userId)`
      - If success: `TempData["Success"] = "Application withdrawn."`
      - If failure: `TempData["Error"] = result.Error`
      - RedirectToAction("MyApplications")
  **verify**: Controller compiles, all actions follow MyAccountController pattern (userId extraction, Challenge(), Result handling)
  **acceptance_criteria**: All 5 actions present. GET Apply pre-fills listingId. POST Apply has ValidateAntiForgeryToken. Withdraw is POST-only. Status messages via TempData
  **read_first**: [src/PetPlatform.Host.MVC/Areas/Customer/Controllers/MyAccountController.cs]
