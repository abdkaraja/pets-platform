    ```csharp
    public interface IAdoptionService

        Task<Result<bool>> CloseListingAsync(int id, string shelterUserId);
        Task<IEnumerable<AdoptionApplicationDto>> GetApplicationsForListingAsync(int listingId, string shelterUserId);

    }
    ```
  **verify**: Interface compiles, uses Result<T> and existing DTOs

- **type**: create
  **files**: [src/PetPlatform.Application/Services/AdoptionService.cs]
  **action**: Implement `AdoptionService` following `PetService.cs` and `OrderService.cs` patterns:

    **Constructor**: Inject `IApplicationDbContext`, `IValidator<CreateListingDto>`, `IValidator<UpdateListingDto>`, `IValidator<CreateApplicationDto>`, `IValidator<ReviewApplicationDto>`. Use `Guard.Against.Null` for each.

    **GetActiveListingsAsync**: Query `AdoptionListings` where Status == Active. Join with `Pets` to flatten pet data. Apply optional filters: Species (exact match on Pet.Species), Breed (Contains on Pet.Breed, case-insensitive), MinAge (Pet.Age >= value), MaxAge (Pet.Age <= value), Size (exact match on Pet.Size), Location (Contains, case-insensitive), SearchTerm (Contains on Title or Description). Apply pagination. Return `PagedResultDto<AdoptionListingDto>`.

    **GetListingByIdAsync**: Find listing by Id with Pet included. Return null if not found. No auth required (public).

    **SubmitApplicationAsync**: Validate DTO. Find listing, verify Status == Active (failure if not). Check for duplicate: no existing application by same ApplicantUserId on same ListingId (failure: "You have already applied for this pet."). Create `AdoptionApplication.Create(listingId, applicantUserId, message)`. Add to context. Save. Return mapped DTO.

    **GetMyApplicationsAsync**: Query applications where ApplicantUserId == userId. Include Listing (and Pet via Listing). Order by CreatedAt desc. Return mapped list.

    **WithdrawApplicationAsync**: Find application by id. Verify ApplicantUserId matches. Verify Status == Submitted (only submitted can be withdrawn). Call `application.UpdateStatus(ApplicationStatus.Withdrawn)`. Save. Return success.

    **GetShelterListingsAsync**: Query listings where ShelterUserId == shelterUserId. Include Pet and count of Applications. Order by CreatedAt desc. Return mapped list.

    **CreateListingAsync**: Validate DTO. Verify Pet exists. Verify no active listing already exists for this PetId (query: PetId == dto.PetId && Status == Active → failure "This pet already has an active listing."). Create via factory. Add to context. Save. Return mapped DTO.

    **UpdateListingAsync**: Validate DTO. Find listing. Verify ShelterUserId matches. Call listing.UpdateDetails(). Save. Return mapped DTO.

    **CloseListingAsync**: Find listing. Verify ShelterUserId matches. Verify Status == Active. Call listing.Close(). Save. Return success.

    **GetApplicationsForListingAsync**: Find listing. Verify ShelterUserId matches. Query applications where ListingId == listingId. Include StatusHistory. Order by CreatedAt desc. Return mapped list.

    **ReviewApplicationAsync**: Validate DTO. Find application. Find listing. Verify ShelterUserId matches listing's ShelterUserId. Verify application.Status is Submitted or UnderReview (cannot review already-decided applications). If new status is UnderReview: call application.UpdateStatus(UnderReview). If new status is Approved or Rejected: call application.Review(reviewerUserId, dto.Status, dto.ReviewNotes). If Approved: also update listing.Status to Adopted. Save. Return mapped DTO.

    **Private helpers**: `MapToListingDto(AdoptionListing listing)` and `MapToApplicationDto(AdoptionApplication application)` static methods following the `MapToDto` pattern from PetService.
  **verify**: Service compiles, all business rules implemented, ownership checks present in every mutation method
  **acceptance_criteria**:
    - Every mutation method has ownership/authorization check
    - SubmitApplication prevents duplicates and requires Active listing
    - CreateListing prevents duplicate active listings per pet
    - ReviewApplication validates reviewer owns the listing
    - Status transitions enforced by domain entity (not re-checked in service)
    - Approving an application also marks listing as Adopted
    - All validators called before domain operations
    - All returns use Result<T> for failure paths
  **read_first**: [src/PetPlatform.Application/Services/PetService.cs, src/PetPlatform.Application/Services/OrderService.cs]
